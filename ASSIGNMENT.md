## Часть 1. Календарь встреч компании

Веб-приложение для планирования встреч внутри команды. Сотрудник добавляет встречу, указывает участников и время - система проверяет конфликты по всем участникам и блокирует бронирование, если кто-то занят.

**Демо:** https://latte-test.onrender.com/login

Данные для входа: `worker` / `worker123`

_Сервис на бесплатном хостинге - может потребоваться 30-60 секунд на первый запуск (холодный старт)._

**Репозиторий:** https://github.com/Huginn41/latte_test

### Стек и почему именно так

**FastAPI** - быстрый старт, автодокументация, встроенная поддержка Pydantic для валидации входных данных. Для такого проекта избыточный фреймворк вроде Django был бы лишним.

**Jinja2 + SSR** - серверный рендеринг без фронтенд-сборщиков. Шаблоны с наследованием (`base.html` -> `extends`) и переиспользуемые частицы через `include`. Vanilla JS только там, где нужен - отправка формы через `fetch` с инлайн-показом конфликта без перезагрузки страницы.

**PostgreSQL + psycopg2** - надёжное хранилище с поддержкой индексов по времени. `RealDictCursor` для удобного доступа к строкам как к словарям.

**Архитектура** - repository pattern: весь SQL только в `repository.py`, роутеры вызывают функции репозитория. Проверка конфликтов через SQL-перекрытие: `start_time < new_end AND end_time > new_start` по всем участникам встречи.

**Деплой** - Render (бесплатный web service) + Supabase (бесплатный PostgreSQL).

## Часть 2. Автоматизация процесса в fashion e-commerce

### Автоматизация ответов на отзывы покупателей

**Что и почему**

В fashion e-commerce отзывы - прямой инструмент продаж. Покупатели читают ответы бренда перед покупкой, а без ответа негативный отзыв висит как красный флаг. При ассортименте от нескольких тысяч позиций ручная обработка отзывов либо съедает время контент-менеджеров, либо большинство отзывов остаётся без ответа вообще.

Идея: GPT-4o принимает текст отзыва + карточку товара (название, категория, ключевые характеристики) и генерирует ответ в голосе бренда. Негативные отзывы с упоминанием брака или несоответствия размеру уходят отдельным потоком - там нужен живой человек.

**Что нужно**

Данные:
- выгрузка отзывов из маркетплейсов (Wildberries, OZON API) и с сайта
- карточки товаров с описаниями
- 200-300 одобренных вручную ответов как примеры tone of voice

Инструменты:
- Claude API или GPT-4o для генерации - берём модель с большим контекстом, чтобы передать примеры голоса бренда прямо в промпт
- простой классификатор для отделения нейтральных/позитивных отзывов от тех, где нужен человек (можно тем же LLM одним запросом)
- очередь задач (например, Celery или простой cron) для асинхронной обработки
- интерфейс модерации - перед публикацией менеджер видит сгенерированный ответ и одним кликом отправляет или правит

**Как проверить, что работает**

Первые 2 недели - режим теневого запуска: система генерирует ответы, но публикует их только после одобрения. Менеджер отмечает каждый ответ: "отправил как есть", "отредактировал", "написал сам". Если "как есть" - больше 70%, система готова к боевому режиму.

**Метрика через месяц**

Основная - доля отзывов с ответом (target: с текущих ~20-30% до 80%+). Вторичная - время первого ответа (target: с нескольких дней до нескольких часов). Дополнительно: рейтинг карточек товаров до/после - отвеченные негативные отзывы часто не снижают конверсию так сильно, как отвеченные без ответа.

### 2. Tilda - VK Webhook

**Что это:** Сервис для получения заявок с сайта на Tilda в личные сообщения VK.

Задача возникла конкретная: нужно было принимать заявки в VK, потому что Telegram тогда активно блокировался, а нативного решения у Tilda не было. Написал за вечер - принимает POST от Tilda, форматирует заказ в читаемое сообщение и отправляет нескольким получателям через VK API.

**Репозиторий:** https://github.com/Huginn41/tilda-vk-webhook

**Роль:** Сделал полностью сам.

**Стек:** FastAPI, VK API, Docker, деплой на Railway.
